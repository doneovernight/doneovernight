# DON Character System

Status: Official foundation, Phase 1

Version: `1.0.0-spec`

Owner: DONEOVERNIGHT

Scope: Permanent brand asset

DON is the permanent DONEOVERNIGHT assistant: a small AI operator that helps, guides, and then gets out of the way.

This directory defines DON with the same governance standard as the DONEOVERNIGHT logo. It is the source of truth for future character drawing, sprite production, animation, product integration, documentation, and interactive demonstrations.

Phase 1 is specification only. It contains no website functionality, runtime component, chat integration, placeholder implementation, or production sprite sheet.

## Canonical documents

- [`DON-CHARACTER-BIBLE.md`](DON-CHARACTER-BIBLE.md) - identity, role, construction, states, scale, and governance.
- [`DESIGN-RULES.md`](DESIGN-RULES.md) - exact pixel geometry, palette, lighting, shadows, expressions, and visual prohibitions.
- [`MOTION-GUIDE.md`](MOTION-GUIDE.md) - frame counts, timing, sequencing, transition, stacking, and reduced-motion rules.
- [`VOICE.md`](VOICE.md) - personality, language, behavioral boundaries, and example messages.
- [`ASSETS/`](ASSETS/) - future-ready asset taxonomy, naming, exports, atlases, and manifests.

## Permanent principles

1. DON is useful before expressive.
2. DON is calm before cute.
3. DON never interrupts.
4. DON never sells.
5. DON never competes with content or controls.
6. DON uses the same silhouette, face, palette, and proportions in every DONEOVERNIGHT product.
7. Product teams choose a state and scale; they do not redraw or restyle DON.

## Product scope

The system is intended for:

- `doneovernight.com`
- HQ
- Client Workspace
- Operator Workspace
- future mobile apps
- documentation
- interactive demos

## Approval boundary

Changes to any of the following require a new version and explicit brand approval:

- silhouette;
- master grid;
- head-to-body ratio;
- face geometry;
- eye geometry or color;
- arms, legs, or feet;
- palette;
- official states;
- animation timing;
- voice principles;
- naming or export conventions.

No product-specific visual variants are permitted without being added to this system first.
